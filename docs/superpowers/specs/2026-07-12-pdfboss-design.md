# pdfboss — design spec (2026-07-12)

A PDF toolkit written from scratch in Rust against the PDF specification (ISO 32000-1/-2):
parsing, text extraction, and page rasterization, with a CLI and Python bindings
(maturin/PyO3), packaged and released exactly like the `konan` repository.

## Ground rules for all contributors (human or agent)

1. **Original work only.** Implement from the ISO 32000 PDF specification. Do not copy,
   port, translate, or reference the source code, identifiers, comments, or documentation
   of any existing PDF library, in any language. Never name other PDF libraries anywhere —
   not in code, comments, docs, README, tests, or commit messages.
2. **Shared build cache.** All work happens in this single checkout with the single shared
   `target/` directory. Never create per-agent worktrees or clones for Rust builds, never
   set `CARGO_TARGET_DIR`, never run `cargo clean`, and build debug-only locally (no
   `--release`). Cargo's own lock serializes concurrent builds; that is fine.
3. **Lenient reader philosophy.** Real-world PDFs are malformed. Parsing is
   best-effort-with-recovery: prefer skipping/defaulting over erroring once the document
   skeleton is loadable. Hard errors are for: unreadable xref after recovery scan,
   encryption, and I/O.
4. **Dependency allowlist.** core: `thiserror`, `flate2`, `memchr`. render: + `png`,
   `jpeg-decoder`. cli: + `clap` (derive). py: + `pyo3` (0.25, `abi3-py312`).
   testkit/text: no external deps. dev-deps free within reason. Nothing else without a
   spec change.
5. Edition 2021, `rustfmt.toml` = `edition = "2021"`, clippy `-D warnings` clean.

## v0.1 scope

**In:** classic xref tables + xref streams + hybrid + `/Prev` chains + recovery scan;
object streams; filters FlateDecode, LZWDecode, ASCIIHexDecode, ASCII85Decode,
RunLengthDecode (+ PNG/TIFF predictors); DCTDecode passthrough (decoded at image layer);
page tree with attribute inheritance; document metadata; content-stream operator parsing
incl. inline images; text extraction (simple fonts with /Encoding + /Differences, Type0
CID fonts via ToUnicode, positional layout); rasterization to RGBA8 PNG (paths,
fill/stroke, even-odd + nonzero, transforms, clipping via mask, DeviceGray/RGB/CMYK +
Indexed color, image XObjects incl. JPEG and image masks, form XObjects, constant alpha);
CLI (`info`, `text`, `render`, `obj`); Python bindings.

**Out (documented limitations, error or skip gracefully):** encrypted files
(`Error::Encrypted`), JPXDecode/Crypt filters, glyph rasterization (text is tracked but
not painted in v0.1), shading/patterns (skipped; pattern fills paint mid-gray), blend
modes beyond Normal, tagged-PDF structure, annotations/AcroForm rendering, digital
signatures, incremental writing (this is a reader).

### lexer.rs
Byte-level tokenizer, zero-copy where possible.
```rust
pub enum Token { Int(i64), Real(f64), Name(Name), LitString(Vec<u8>), HexString(Vec<u8>),
                 ArrayOpen, ArrayClose, DictOpen, DictClose, Keyword(Vec<u8>), Eof }
pub struct Lexer<'a> { /* data: &'a [u8], pos: usize */ }
impl<'a> Lexer<'a> {
    pub fn new(data: &'a [u8]) -> Self;
    pub fn at(data: &'a [u8], pos: usize) -> Self;
    pub fn pos(&self) -> usize;  pub fn seek(&mut self, pos: usize);
    pub fn next_token(&mut self) -> Result<Token>;
    pub fn peek_token(&mut self) -> Result<Token>;
    pub fn skip_whitespace_and_comments(&mut self);
    pub fn data(&self) -> &'a [u8];
}
```
Rules (ISO 32000 §7.2/7.3): whitespace = NUL/HT/LF/FF/CR/SP; delimiters `()<>[]{}/%`;
comments `%`→EOL; numbers may lead with `+ - .` (lenient: `--5` → 0? no — treat repeated
signs as error-tolerant: parse first sign, rest as number; `.` alone = 0.0); names decode
`#xx` (bad hex kept literal); literal strings balance nested unescaped parens, escapes
`\n \r \t \b \f \\ \( \)`, 1–3 digit octal, backslash-EOL line continuation, raw
EOL→`\n`; hex strings ignore whitespace, odd length pads trailing `0`; everything else
regular → `Keyword` (incl. `true false null obj endobj stream endstream R xref trailer
startxref n f`).

### parser.rs
```rust
pub trait Resolve { fn resolve_ref(&self, r: ObjRef) -> Option<Object>; }
pub struct NoResolve;                                  // impl Resolve, returns None
pub struct Parser<'a> { /* lexer */ }
impl<'a> Parser<'a> {
    pub fn new(data: &'a [u8]) -> Self;  pub fn at(data: &'a [u8], pos: usize) -> Self;
    pub fn pos(&self) -> usize;  pub fn seek(&mut self, pos: usize);
    pub fn parse_object(&mut self, resolver: &dyn Resolve) -> Result<Object>;
    /// Expects `N G obj … endobj` at current pos; returns (ref, object).
    pub fn parse_indirect(&mut self, resolver: &dyn Resolve) -> Result<(ObjRef, Object)>;
}
```
`int int R` → `Object::Ref` via lookahead with backtrack. Streams: after `stream` +
EOL, take `/Length` (resolving via `resolver` if it's a Ref); if missing/wrong, recover
by scanning for the nearest `endstream` (memchr) and trimming a trailing EOL. Lenient:
`endobj` missing → accept at next `obj`/EOF.

### filters/mod.rs
```rust
pub fn decode_stream(stream: &Stream, resolver: &dyn Resolve) -> Result<Vec<u8>>;
```
Reads `/Filter` (name or array) + `/DecodeParms` (dict or array, `null` ok; also accept
`/DP`, `/F` abbreviation only for inline images — see content.rs), applies chain in
order. Individual `pub fn`s in submodules, each `(&[u8], params) -> Result<Vec<u8>>`:
- `flate`: via `flate2`; tolerate junk after stream end and truncated data (return what
  decoded). Then predictor post-pass.
- `lzw`: own implementation — 9→12-bit codes, clear=256, EOD=257, `EarlyChange`
  default 1. Then predictor post-pass.
- `ascii_hex`: whitespace ignored, `>` ends, odd digit padded with `0`.
- `ascii85`: whitespace ignored, `z` = 4 zero bytes (only between groups), `~>` ends,
  final partial group of n chars → n−1 bytes. Leading `<~` tolerated.
- `run_length`: length byte 0–127 → copy n+1; 129–255 → repeat next byte 257−n; 128 EOD.
- `predictor(data, predictor, colors, bpc, columns)`: 2 = TIFF horizontal diff (8-bit
  components only, else pass through); ≥10 = PNG filters per row (None/Sub/Up/Avg/Paeth),
  `bpp = max(1, colors*bpc/8)`, row length = `ceil(colors*bpc*columns/8)`.
- `DCTDecode`/`JPXDecode`: **passthrough** (returned raw; image layer handles JPEG). If a
  passthrough filter is not last in the chain → `UnsupportedFilter`.
- `Crypt` → `UnsupportedFilter`.

### xref.rs + objstm.rs
```rust
pub enum XrefEntry { Free, InFile { offset: u64, gen: u16 }, InStream { stream_num: u32, index: u32 } }
pub struct Xref { /* map u32 → XrefEntry */ pub trailer: Dict }
pub fn load_xref(data: &[u8]) -> Result<Xref>;
```
`load_xref`: find `startxref` in last 1 KiB (scan back further if absent, up to 64 KiB);
parse section at offset — `xref` keyword ⇒ classic subsections `start count` + 20-byte
entries (lenient on 19/21-byte lines), then `trailer` dict; else ⇒ xref **stream**
(`/Type/XRef`, `/W [a b c]` with 0-width type field defaulting to 1, `/Index` defaulting
`[0 Size]`, big-endian fields, stream's own dict is the trailer). Follow `/Prev` chain
(oldest last — earlier sections win only where newer lacks the entry: first-seen wins
walking newest→oldest); classic trailers may carry `/XRefStm` (hybrid) — merge it before
`/Prev`. Loop-guard visited offsets. On any failure: **recovery scan** — walk the whole
file for `N G obj` headers (build table, last occurrence of an obj num wins), find a
trailer dict or any `/Type/Catalog` object for `/Root`. `Xref::merge`, `Xref::get`.
objstm.rs: `pub fn extract(stream_data: &[u8], n: usize, first: usize, index: u32) ->
Result<Object>` — header = `2*n` ints (objnum offset)…, offsets relative to `first`.

## pdfboss-text

```rust
pub fn extract_text(doc: &Document, page: &Page) -> Result<String>;
pub struct TextSpan { pub text: String, pub x: f32, pub y: f32, pub size: f32, pub font: String }
pub fn extract_spans(doc: &Document, page: &Page) -> Result<Vec<TextSpan>>;
```
Internal modules:
- `font.rs`: `Font { simple: bool, code_to_unicode, widths, default_width, space_code }`
  loaded from a page `/Font` resource dict. Simple fonts (Type1/TrueType/Type3):
  byte codes; unicode priority = `/ToUnicode` CMap > `/Encoding` base
  (`WinAnsiEncoding`/`MacRomanEncoding`/`StandardEncoding` tables, hand-built from spec
  Appendix D) + `/Differences` (glyph name → unicode via bundled AGL subset:
  ASCII names, `uniXXXX`, `uXXXX[XX]`, common ~250 names) > Standard fallback. Widths
  from `/FirstChar`+`/Widths`, missing → `/FontDescriptor` `/MissingWidth` → 500.
  Type0/CID: 2-byte codes (assume Identity-H/-V or CMap name ending in `-H/-V`; other
  CMaps treated as 2-byte), unicode via `/ToUnicode` only (else U+FFFD), widths from
  descendant `/W` + `/DW` (default 1000).
- `cmap.rs`: ToUnicode parser — `begincodespacerange`, `beginbfchar` (`<src> <dst>`),
  `beginbfrange` (`<lo> <hi> <dst>` incrementing last UTF-16 code unit, and
  `<lo> <hi> [<dst>…]` array form). Dst hex = UTF-16BE, may be multi-char.
- `encodings.rs`: the three base tables + AGL subset.
- `extract.rs`: runs `Vec<Op>` with text state (Tm/Tlm, Tf/Tfs, Tc, Tw, Tz, TL, Ts,
  Trise); per shown string: split into codes (1 or 2 byte), advance
  `(w/1000·Tfs + Tc + Tw·[code==32, 1-byte only])·Tz/100`; TJ numbers subtract
  `n/1000·Tfs·Tz/100`. Span origin = Tm·CTM applied to (0, Ts). Emits spans; Form
  XObjects recursed (depth cap 16) with their own /Resources falling back to page's.
  Layout pass: group spans into lines when |Δbaseline_y| ≤ 0.5·size, order lines
  top→bottom (device space: y desc), spans left→right; insert `" "` when horizontal gap
  > 0.25·size; join lines with `\n`. Text-render-mode 3 (invisible) **is** extracted.

## pdfboss-render

```rust
pub struct Pixmap { pub width: u32, pub height: u32, pub data: Vec<u8> /* RGBA8, straight alpha, row-major */ }
impl Pixmap { pub fn new(w: u32, h: u32) -> Pixmap;         // transparent
              pub fn fill(&mut self, rgba: [u8;4]);
              pub fn encode_png(&self) -> Result<Vec<u8>>;
              pub fn save_png(&self, path: impl AsRef<Path>) -> Result<()>; }
pub fn render_page(doc: &Document, page: &Page, scale: f32) -> Result<Pixmap>;
```
`render_page`: pixel size = `ceil(crop_w·scale) × ceil(crop_h·scale)` (post-rotation),
white background, base CTM maps crop box → device with y-flip and /Rotate applied.
Internal modules:
- `path.rs`: `PathBuilder` (move/line/cubic/close/rect) → flattened polygons
  (adaptive cubic subdivision, tolerance 0.1px, hard cap 2^10 segments/curve).
- `raster.rs`: scanline coverage rasterizer — accumulate per-pixel coverage from edges
  (analytic horizontal coverage, 4× vertical subsampling is acceptable), fill rules
  nonzero + even-odd, output = `fn fill_path(pix, polys, rule, rgba, alpha, clip: Option<&Mask>)`.
  `Mask` = u8 coverage buffer, same dims as pixmap; clip intersection = min().
- `stroke.rs`: flatten, then each segment → offset quad of width w (min 0.75px device);
  joins/caps approximated round via small fans; dash patterns applied at flatten level
  (segment splitting).
- `color.rs`: `ColorSpace { DeviceGray, DeviceRGB, DeviceCMYK, Indexed{base, lookup},
  Other(n_components) }` parsed from resources (`ICCBased` → by `/N`; `CalRGB/CalGray/Lab`
  → Device equivalents; `Separation/DeviceN` → treat as gray via tint transform *skipped*:
  approximate tint 1−v). `to_rgb(&[f32]) -> [f32;3]`; CMYK naive: `1−min(1, x+k)`.
- `image.rs`: decode image XObject / inline image → RGBA buffer: bpc 1/2/4/8/16(→8),
  `/Decode` arrays, `/ImageMask` stencils (paint fill color through mask), DCTDecode via
  `jpeg-decoder` (gray/RGB/CMYK incl. Adobe inverted-CMYK heuristic), Indexed lookup.
  Draw: inverse-map each device pixel in the unit-square image quad, nearest-neighbor
  sample, composite source-over.
- `executor.rs`: `GState { ctm, stroke/fill color+space, line width/cap/join/dash,
  alpha ca/CA, clip: Option<Mask> }`, stack for q/Q (depth cap 64); runs `Vec<Op>`;
  W/W* set pending clip applied after next paint op; `gs` ExtGState honors `/ca /CA /LW
  /LC /LJ /D`; `Do` form XObjects: recurse (depth 16, own resources fallback page,
  `/Matrix`, `/BBox` clip); `sh`/patterns: skip; text ops: tracked, glyphs not painted.
  All numeric input clamped to finite (NaN/Inf → op skipped).

## pdfboss-cli

clap derive, binary `pdfboss`:
- `pdfboss info <file>` — version, page count, page sizes, metadata table, encrypted flag.
- `pdfboss text <file> [--page N]` — 1-based page, default all pages separated by `\f`.
- `pdfboss render <file> --page N [-o out.png] [--scale S]` — default out `page-N.png`, scale 1.0.
- `pdfboss obj <file> <num> [gen]` — pretty-print the object (decoded stream dicts, data length note).
Exit 1 with message on stderr for errors.

## Testing strategy

- Every module: `#[cfg(test)]` unit tests written **with or before** implementation
  (lexer tokens incl. all escape forms; each filter incl. edge cases — LZW EarlyChange,
  ascii85 `z`/partial groups, PNG Paeth; predictor rows; xref classic+stream+prev+hybrid+
  recovery from corrupt offsets; objstm; page-tree inheritance/cycles; content ops incl.
  inline images; CMap bfrange forms; extraction spacing; matrix math; rasterizer:
  axis-aligned rect exact coverage, triangle winding, even-odd donut, clip intersect;
  image decode bpc/Decode; CMYK conversion).
- Integration tests use `pdfboss-testkit` (dev-dependency) — never hand-computed offsets
  in test bodies.
- pytest for the Python surface (see above).
- Gate: `cargo fmt --check`, `cargo clippy --workspace --all-targets -- -D warnings`,
  `cargo test --workspace`, `maturin develop` + `pytest` locally via uv.
